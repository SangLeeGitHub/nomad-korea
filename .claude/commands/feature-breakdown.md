$ARGUMENTS

## 분석 프로세스

### 1단계: 문제 정의
- 최종 목표가 무엇인가?
- 현재 상태는 어떠한가?
- 제약 조건과 범위는 무엇인가?

### 2단계: 분해 기준 적용
다음 기준을 순차적으로 적용하여 분해하세요:

1. **기능별 분해**: 독립적으로 동작하는 기능 단위로 분리
2. **계층별 분해**: UI → 비즈니스 로직 → 데이터 레이어
3. **의존성 분석**: 선행 작업과 후행 작업 식별
4. **복잡도 평가**: 핵심 기능과 부가 기능 구분

### 3단계: 하위 작업 검증
각 하위 작업이 다음 조건을 만족하는지 확인:
- [ ] **독립성**: 다른 작업 완료 없이 시작 가능
- [ ] **검증 가능성**: 완료 여부를 명확히 확인 가능
- [ ] **적절한 크기**: 한 번에 집중할 수 있는 범위
- [ ] **명확한 입출력**: 입력과 산출물이 정의됨
