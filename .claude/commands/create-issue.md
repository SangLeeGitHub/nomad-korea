# GitHub 이슈 생성

당신은 GitHub 이슈를 체계적으로 작성하는 전문가입니다. 아래 내용을 바탕으로 GitHub 이슈를 생성해주세요.

## 생성할 이슈 내용

$ARGUMENTS

---

## 이슈 생성 프로세스

### 1단계: 이슈 분석
- 이슈 유형 파악 (bug, feature, enhancement, documentation, refactor 등)
- 핵심 요구사항 정리
- 관련 컴포넌트/파일 식별

### 2단계: 이슈 구조화
다음 정보를 정리하세요:
1. **제목**: 명확하고 간결한 이슈 제목
2. **유형**: bug | feature | enhancement | documentation | refactor
3. **설명**: 이슈의 상세 내용
4. **작업 항목**: 체크리스트 형태의 세부 작업

### 3단계: 라벨 및 우선순위 결정
- 적절한 라벨 선택
- 우선순위 평가 (high, medium, low)

---

## 실행

위 분석을 바탕으로 `gh issue create` 명령어를 사용하여 GitHub 이슈를 생성하세요.

### 이슈 형식

```markdown
## 개요
[이슈에 대한 간략한 설명]

## 상세 내용
[구체적인 요구사항 또는 문제 설명]

## 작업 항목
- [ ] 작업 1
- [ ] 작업 2
- [ ] 작업 3

## 참고 사항
[관련 파일, 의존성, 주의사항 등]
```

### 명령어 예시

```bash
gh issue create --title "이슈 제목" --body "$(cat <<'EOF'
## 개요
...

## 상세 내용
...

## 작업 항목
- [ ] ...

## 참고 사항
...
EOF
)" --label "feature"
```

---

## 주의사항

- 이슈 제목은 50자 이내로 명확하게 작성
- 본문은 한국어로 작성
- 라벨이 존재하지 않으면 라벨 옵션 생략
- 이슈 생성 후 URL을 사용자에게 제공

이제 위 내용을 바탕으로 GitHub 이슈를 생성해주세요.
